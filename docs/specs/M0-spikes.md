# M0: リスク退治スパイク

ステータス: **確定**

## 目的(退治する落とし穴)

- A-1: WebView↔ネイティブ描画ブリッジ(単一最大リスク)
- B-2: 動画デコード(シーク・VFR)
- B-1: 時間表現と再生クロック

スパイクは**使い捨てコード**。品質・構造は不問、`spikes/`ディレクトリに隔離し、本体クレートには一切取り込まない。成果物はコードではなく**実測値と採否判断の記録**(`docs/spikes/`に1スパイク1レポート)。

## スコープ外

- 本体のクレート設計、プラグインtrait、キャッシュ、3D、音声実装(時間表現の設計のみ行い、音声再生はやらない)

| ID | 内容 | 依存 | 完了条件 |

| S1b | ブリッジ方式(b): wgpuレンダ→`copy_texture_to_buffer`読み戻し→カスタムURIスキーム(またはローカルWebSocket)→WebView `<canvas>`描画。1080p30と4K30で実測 | なし | `docs/spikes/s1b-readback.md` にフレームタイム・E2Eレイテンシ・CPU/GPU使用率を記録 |

## 並列レーン

S1a / S1b / S2 / S3 は**全て独立・同時着手可**。S1cのみ合流点。

## S1cの判断基準(事前固定)

| 基準 | 合格ライン |
|---|---|
| プレビュー描画 | 1080p30を落とさず表示 |
| UI共存 | モーダル/ドロップダウン/パネルリサイズがプレビュー領域と重なって正常動作 |
| スクラブ | シークバー操作への追従レイテンシが体感即応(目安100ms以内) |
| 実装コスト | OS固有コードが1プラットフォーム分で済む(v1は単一OS方針) |

(b)が合格ならシンプルさを優先して(b)を採用。(b)不合格かつ(a)合格なら(a)。両方不合格の場合のみGPUI等のフルネイティブUI(方式(d))の検討に進み、M3仕様を書き直す。

## フェーズ完了条件

4本のスパイクレポートが揃い、S1c判断・S2クレート採否・S3時間型がM1/M3仕様書に反映されていること。
