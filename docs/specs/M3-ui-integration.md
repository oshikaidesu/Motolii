# M3: UI統合

ステータス: **ドラフト**(M0-S1cの採否判断とM2ドキュメントモデル確定を受けて確定)

## 目的(退治する落とし穴)

A-1(ブリッジ方式の最終確定)、D-3(OpenCut流用の期待値管理)。

## 方針

- プレビューブリッジはM0-S1cで採択した方式を実装(この節はS1c後に書き換える)
- UIはTauri + React。OpenCut(MIT)はコード丸ごと流用ではなく**操作仕様+コンポーネント単位のつまみ食い**(references.md参照)
- タイムラインUIの状態管理はM2ドキュメントモデルに直結(UI独自の編集状態を二重に持たない)。編集操作は全てM2コマンドを発行する形
- キーフレーム編集UIの操作仕様はTheatre.js(シーケンス/グラフエディタ)を参考(AGPLのstudioコードは読まない・流用しない)

## タスク分割(粗案)

| ID | 内容 | 依存 | 完了条件(概要) |
|---|---|---|---|
| U1 | Tauriシェル + プレビューブリッジ(S1c方式)実装 | S1c, M2-D5 | 1080p30プレビューがUI内で表示、S1c判断基準を本実装で再達成 |
| U2 | IPC層: ドキュメントコマンド送受(UI→コア)+ 状態購読(コア→UI) | M2-D2 | UIからの編集がUndo/Redo込みで往復するテスト |
| U3 | タイムラインUI(トラック/クリップ配置、ドラッグ、トリム、波形表示) | U2 | マウスでクリップ配置・移動・トリムができる |
| U4 | パラメータパネル + キーフレーム編集(値のキー打ち、補間切替) | U2 | キーフレーム編集がプレビューに反映される |

並列レーン: U1 と U2 は独立着手可。U3/U4/U6はU2後に並列。U7はU3後。
