# M4: キャッシュ層と解析の拡充

ステータス: **ドラフト**(凍結ゲートで確定)

## 目的(退治する落とし穴)

B-5(時系列依存とキャッシュ)、B-6(OpenCV依存)、F-2(キャッシュの並行契約 — Natronがデッドロックした正確なポイント)。

## 方針

- キャッシュキー = ノードID × 時間区間 × 入力パラメータハッシュ。時系列解析ノードは**区間単位**のキャッシュエントリ(フレーム単位ダーティフラグでは表現しない)
- RAM/VRAM別のメモリ予算とLRU追い出しをキャッシュ層の初期設計に含める(4K RGBA=33MB/枚を常に意識)
- **並行契約(F-2、K1の設計に含める)**: (1) 読み手はエントリのハンドル(参照カウント)を取得して使う。追い出しは「LRU選定→参照ゼロになってから解放」の遅延方式で、**使用中エントリを無効化しない**。(2) キャッシュの内部ロックは1段のみ(ロック保持中に別のロックやGPU同期を取らない)。(3) 無効化(パラメータ変更)はwriterのコマンド適用起点でエポック/世代番号を進め、古い世代のスナップショットで走行中のレンダは自分の世代のエントリを読み切ってよい。(4) バックグラウンドのプロキシ/解析ジョブ(K4/K5)はスナップショット読み+結果メッセージ返しのみで、キャッシュ/ドキュメントへ直接書かない(M2-D8の所有権モデルに従う)
- インポート時パイプライン: プロキシ生成(低解像度)+ VFR→CFR正規化
- 色解析(支配色/色マスク重心)や高度な解析(オプティカルフロー等)は、この段階で解析プラグインとして隔離導入。導入前にwgpu computeでの自前実装可否を先に評価し、OpenCV / ONNX Runtime等の外部依存は必要になるまで入れない
- ノードグラフの領域伝播(RoI)設計はNatronの考え方を参考(references.md、GPLのため設計参考のみ)

## タスク分割(粗案)

| ID | 内容 | 依存 | 完了条件(概要) |
|---|---|---|---|
| K1 | キャッシュ層(キー設計、RAM/VRAM予算、LRU、計測フック、**並行契約(参照カウントハンドル+遅延解放+単段ロック)**) | 凍結ゲート, M2-D8 | 予算超過時の追い出しと再計算範囲のユニットテスト。**読み手が使用中のエントリが追い出されないこと・読み手と追い出しの並行実行でデッドロックしないことのストレステスト(loomまたはマルチスレッド反復)** |
| K2 | oc-render統合: パラメータ変更時の無効化伝播 | K1 | 「1パラメータ変更→影響ノードのみ再計算」をプロファイルで確認 |
| K3 | 時系列解析ノードの区間キャッシュ + 部分再解析 | K1 | 区間の一部変更で全区間再解析にならないテスト |
| K4 | インポートパイプライン(プロキシ生成、CFR正規化、バックグラウンド実行) | M2-D1 | VFRスマホ素材でフレームズレなし、4K素材でプロキシスクラブが破綻しない |
| K5 | 解析プラグイン: 色解析(支配色/色マスク重心)を最初の実装候補にし、必要ならオプティカルフローも評価・選定 | K3 | 既知の動き/色のテスト素材で精度検証し、DataTrackとして評価エンジンから参照可能 |
| K6 | SVGインポート(usvg)→ベクターソースノード(Vello描画)。コンセプト決定によりコア機能 | M1-T7(Vello採用) | 代表的なSVG(パス・グループ・塗り/線)のゴールデンイメージテスト、アセットとしてタイムライン配置可能 |
| K7 | グループ仮出力(ベイク): 選択グループの出力を時間範囲でキャッシュ(プリコンポ代替、コンセプト決定)。グループ内の編集で自動無効化 | K1, M2-D1 | ベイク後のスクラブがグループ再計算なしで行われ、グループ内編集で正しく無効化されるテスト |

並列レーン: K1→K2→K3 と K4 は独立。K5はK3後。K6は独立(UI接続はM3-U6と調整)。
