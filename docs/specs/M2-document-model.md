# M2: ドキュメントモデルとタイムライン

ステータス: **ドラフト**(凍結ゲートで確定)

## 目的(退治する落とし穴)

C-1(Undo後付け)、C-2(スキーマ進化)、B-1(音声・時間表現)。

## 方針

- プロジェクト状態は単一のserde可能な純データ構造(ドキュメントモデル)。エンジン(oc-render)はこれを読むだけ
- 全編集操作はコマンド(適用/逆適用可能な差分)として実装 → Undo/Redoはコマンド履歴で自動的に得られる
- ファイル形式はバージョンフィールド付きJSON + マイグレーション関数の枠組み
- 再生の主クロックは音声(cpal出力コールバック駆動)。映像は音声クロックに追従

## タスク分割(粗案。凍結ゲートで1PR粒度に再分割)

| ID | 内容 | 依存 | 完了条件(概要) |
|---|---|---|---|
| D1 | oc-doc: ドキュメントモデル(プロジェクト/トラック/クリップ/プラグインインスタンス/キーフレーム)+ serde + バージョン/マイグレーション枠組み | 凍結ゲート | 保存→読み込み→同一レンダ結果のroundtripテスト |
| D2 | コマンドシステム(apply/revert)+ Undo/Redo履歴 | D1 | 全編集コマンドに対し「apply→revert→状態一致」のプロパティテスト |
| D3 | ドキュメント→レンダグラフ変換(oc-docとoc-renderの接続層) | D1 | M1のE2Eテストがドキュメントモデル経由で通る |
| D4 | oc-audio: Symphoniaで音声デコード + cpal出力 + 音声主クロック | 凍結ゲート | 10分素材でA/Vドリフトなし(実測) |
| D5 | プレビュー再生ループ(音声クロック追従、フレーム落とし方針) | D3, D4 | スクラブ/再生/一時停止がCLIまたは最小ウィンドウで動作 |
| D6 | 書き出しへの音声mux(oc-media Encoderに音声入力を追加、映像+音声のmp4を出力) | D4 | 書き出したmp4の音声が元素材とサンプル一致し、映像とズレない(コンセプト: MVの最終書き出しに必須) |

並列レーン: D1→D2→D3 と D4 は独立。D5が合流点。D6はD4後に独立で進められる。
