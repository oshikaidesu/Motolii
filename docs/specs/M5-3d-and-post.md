# M5: 3D合成とポストプロセス

ステータス: **ドラフト**(凍結ゲートで確定)

## 目的(退治する落とし穴)

C-4(2.5Dとブレンドモードの衝突)。

## 方針(discussion-log 6のスコープをC-4の割り切りで具体化)

- シーングラフは2D/3D統一。ただし**合成順序はレイヤー(タイムライン)順が正、Zはカメラ投影(変形)にのみ使う**。レイヤー間の深度合成はしない。メッシュ内部の深度のみ3Dパス内でZバッファ解決
- カメラは単一(位置・注視点・FOV)。ライティングはunlit(必要になったら固定1灯)
- OBJはglTF変換パスで受ける(内部はglTFのみ)
- 被写界深度はZ距離ベースのポストブラーで代用
- ポストプロセスは通常のRenderNodeとして実装(専用機構を作らない)

## タスク分割(粗案)

| ID | 内容 | 依存 | 完了条件(概要) |
|---|---|---|---|
| P1 | glTF読み込み(メッシュ・マテリアル最小限)+ OBJ→glTF変換パス | 凍結ゲート | サンプルアセットの読み込みゴールデンテスト |
| P2 | 3DレイヤーNode: メッシュ/動画テクスチャ平面をカメラ投影でレンダ(レイヤー単位に1パス) | P1 | Zを持つ平面レイヤーがカメラ移動で視差するゴールデンテスト |
| P3 | カメラのドキュメントモデル統合(キーフレーム可能なカメラパラメータ) | P2, M2-D1 | カメラアニメーションのE2Eテスト |
| P4 | ポストプロセスNode群: ブラー(+Z距離マスク)、色調整(リフト/ガンマ/ゲイン)、グレイン | 凍結ゲート | 各ノードのゴールデンテスト |
| P5 | 統合検証: 動画平面 + glTFメッシュ + トラッキング駆動オーバーレイ + ポストを1シーンで書き出し | P3, P4 | E2Eゴールデンテスト + 実素材でのショット制作 |

並列レーン: P1→P2→P3 と P4 は独立。P5が合流点。
