第2案の変更点 — 第1案と同一fixture・同一token role

比較対象は構成ではなくspacing規律。色role・fixture・機能badgeの意味は第1案(m3-main-ui-v1.html)と同じ。

  • spacing token化: 全padding/gap/行高/radiusを`--sp-*`/`--row-*`/`--radius`から取得。raw値の場当たり指定なし(機械検査可能)
  • 罫線+明度による分離: カード・影・空白分離を廃止。sectionは`--inset`地の見出し+罫線で区切る
  • asset browserを行型へ: カード型を廃止し22px行。同面積で6アセット+状態chip(要再リンク=error)を一覧
  • timeline 10行常視: text / mesh / imageの項目種別色を追加収載し、item roleの全種を1画面で確認可能
  • status bar追加: Blender式の文脈ヘルプ(hover対象・操作・shortcut)と実行時statを常設。ヘルプはこの位置の口だけが規約
  • 未解決のまま残すもの: glyphはemoji/文字のplaceholder。hue近接(solo黄×warning琥珀等)はG0-6のCVD測定で再配置
