バッジ対応表 — どの決定を写したか

ステータスは 決定済み=conceptに昇格済み / 方向確定=ux-frontierで方向あり / 未決=想像=プロトタイプ比較の対象。第1案の変更点: Ableton語彙(フラット・角2px・色=機能)、設定画面でDark既定+Light選択、custom themeを妨げない全色トークン化、1 role 1意味のtoken分離(選択/キーフレーム/音楽時間/配線/有効/キャッシュ/solo/mute/warning/error)。

1
アセットブラウザ 決定済み

アプリ内エクスプローラーはコアUI。楽曲は「♪楽曲」チップで1本だけ特別扱い(音声=楽曲1本の決定)。

concept.md / M3-U6
2
1本のエフェクトスタック 決定済み

パス演算子(ベクター域)→ ラスタライズ境界(可視)→ Filter(ピクセル域)を単一の順序リストで表示。二重体系を見せない。

concept.md 2026-07-10 / M3 / F-8・F-13
3
ドライバ=一級オブジェクト 方向確定

Ableton LFO/エンベロープフォロワーの翻訳。1ドライバ→N配線、「マップ」ボタン、帯域別音量フォロワーは標準搭載。置き場所(コンポ共有パネル)は仮。

ux-frontier U-A
4
区間イージングポップアップ 決定済み

Flow/AM式。2キーフレーム選択→ポップアップ。補間タイプ(ベジェ/バウンス/エラスティック/ステップ=Interp variant)+プリセット+4値、オーバーシュートはyの[0,1]外。グラフビューは作らない。

concept.md / M3-U4 / Interp::Bezier
5
楽曲基準線: 波形+BPMグリッド 決定済み

タイムライン最上段に波形をピン留め。ビート検出はせずBPM手動入力→グリッド+スナップ。ルーラーは小節.拍。

concept.md / M3-U7 / ux-frontier U-F
6
ベイク状態バー 方向確定

DAWフリーズトラックの翻訳。仮出力ONのグループに区間バー: 実線=ベイク済み / 縞アニメ=進行中 / 淡色=ダーティ。解凍操作は無い(編集→自動無効化)。

ux-frontier U-E / M4-K7
7
音楽同期エミッション 未決=想像

バーストをビートグリッドに「ノーツとして置く」(ステップシーケンサの翻訳)。rate=パラメータ/バースト=ノーツの分離。この描写はプロトタイプ2案比較の前提でしかない。

ux-frontier U-D / SIM-5
8
トランスポート+品質切替 決定済み

TCと小節.拍の併記、BPM・スナップ常時表示。ドラフト⇄最終品質は同一 render_frame(t, Quality) のモード切替。

concept.md / performance-model / M3-U5
意図的に描いていないもの: U-C(3Dギズモ・CompCamera操作)は「v1で最難のUX新作・未決」のため2Dバウンディングボックスのみ描画(3Dモードは作らない、が既決)。 ターゲットピッカー(U-B ピックウィップ型付き版)は「リンク」行の口だけ置いた。 ノードグラフ・値グラフエディタ・調整レイヤー・マスクパス編集は存在しないこと自体が仕様。
テーマ実装の含意: このページの切替は「UIコードは色名を知らない(トークンだけ参照)→ テーマ=トークン値の束」の実演。設定はユーザー単位で保存し、Documentへ焼き込まない。Slintではglobal singleton、配布形式はLiveの.ask相当のフラットなファイルで足りる。
