# 決定逆引き台帳 (decision-index)

主題キーワードから既決事項を逆引きするための索引。**内容の正本は各リンク先**であり、ここには1行要旨しか書かない(二重管理をしない)。

制定の経緯(2026-07-19): AM式高度イージング(Bounce/Elastic/Steps=区間補間、2026-07-10決定)が[concept.md](concept.md)に正本化されていたにもかかわらず、主題からの逆引き手段がなく、後続のモック・仕様作業が「未決」と誤認して旧仕様と混在した。決定は書くだけでは足りず、**触る主題から引ける**必要がある。

## 使い方(作業前の義務)

1. 仕様・モック・コードを触る前に、対象の主題キーワードでこの表を検索する(grep可能な語を並べてある)
2. ヒットした行の**正本を読んでから**着手する。この表の要旨だけで設計判断しない
3. 新しい決定・撤回・未統一が生まれたら、正本(concept.md/レビュー文書/仕様書)へ書いた上で、**同じ変更でここに1行追加する**。会話・commit履歴にしか残らない決定を作らない
4. 状態語彙は固定: **決定 / 縮小採用 / 延期 / 棄却 / 撤回 / 未統一 / 観察 / 比較中 / 停止線**(`scripts/check-docs.sh`が検証)

過去決定の全量転記は目的ではない。作業で触れた主題から順に登録する(この表に無いこと≠未決。無ければ[reviews全文書索引](reviews/README.md)と[concept.md](concept.md)を検索する)。
