# M2恒久焼き込みの予防(2026-07-12)

ステータス: **運用手順**(採用分)。先人調査の詳細・判定表は[2026-07-12-rework-prior-art.md](2026-07-12-rework-prior-art.md)。本ファイルは「何をすれば失敗しないか」だけを正本にする。

## 前提

- 入場ゲート達成後も、Documentスキーマへの**早焼き**が出戻りの本丸([入場ゲート](2026-07-11-M2-entry-gate.md)の3軸: 恒久性×並列初陣×検証の弱さ)
- いまの出戻りはまだ最小(D1a〜c完了後のフォローアップ D1g/D1h/D1i)。**この窓が安い間に予防を固める**
- 復旧(migration / Legacy variant / 全書き直し)は副次。Olive/OpenCut/Natronがやったのは失敗後の対応であり、本手順の主目的ではない

## やること(予防5手)

### 1. 意味を先に文書化し、コードはその写しにする

恒久に焼く前に、仕様へ次を揃える。揃うまで型・serde・ゴールデンを増やさない。

| 焼くもの | 先に揃える文書 |
|---|---|
| PathOp / 演算 | variantごとの意味論表 — **文書側済**(M2「PathOp意味論表」)。残=validate+幾何+ゴールデン(**D1i-2実装**) |
| DocParam受け口 | 期待型表 — D1h済み |
| TimeMap / 区間 | 写像式・尺の正本・OverrunMode — D1g済み |
| 色・補間・変形順 | M2E-13 / 補間・変形の意味論宣言 — 済み |

**判定**: 意味論表が無いPRは「実装」ではなく「仕様穴」。実装を止めて仕様改訂PRを先に(H-3)。

### 2. 恒久面を狭く保つ

焼いてよいのは「ユーザーデータとして残るもの」だけ。次は焼かない/まだ仮:

- 未決のUI都合(コマンド粒度、波形ピーク) — M2未決のまま実装デフォルト禁止
- 画素アルゴリズムの「改善」(既存variantの意味変更) — 新variantにするか、意味論ゴールデンが無いなら触らない
- CompCamera等、単位未宣言の席 — 台帳どおり入れない

**判定**: 「便利だからフィールド追加」は却下。席が要るなら仕様に【決定】を足してから。

### 3. 追加的変更を選び、解釈変更を選ばない

| やり方 | いつ使う |
|---|---|
| 新フィールド+default / 新enum variant | 意味を足すとき(前方は`min_reader_version`で拒否し得る) |
| 旧形式拒否 + D1e migration | 形状を入れ替えるとき(D1g/D1i-1型) |
| 既存フィールドの意味の書き換え | **禁止**(キーフレーム済みデータの静かな破壊) |

先例の型: SQLiteは3.0以降フォーマットを壊さない([formatchng](https://sqlite.org/formatchng.html))。AEはアルゴリズム変更を新match nameにする([Obsolete effects](https://helpx.adobe.com/after-effects/using/obsolete-effects.html))。Blenderは破壊を設計タスクの互換節で先に宣言する([handbook](https://developer.blender.org/docs/handbook/guidelines/compatibility_handling_for_blend_files/))。

### 4. 依存直列を飛ばさない

M2並列レーン表が正本。いまの硬い線:

- **D1i-2完了前にD3しない**
- D2はD1i-1後、D1e変換実装はD1i-1後
- スキーマ非接触のD4/D6のみゲート対象外のまま先行可

**判定**: 「並列を増やしたい」は依存表の改訂PRが先。発注の独断変更禁止。

### 5. 「完了」は意味の審判で付ける

`cargo test`緑は必要だが十分ではない。完了にしてよい条件:

1. 上記1の意味文書が仕様にあり、実装がそれに一致するテストがある
2. 縮退・黙殺フォールバックを「仕様違反の拒否」に反転済み(第二監査の教訓)
3. 恒久物なら決定への逆リンクがある

意味論ゴールデン(D1i-2/D1i-4)が審判。更新で通すのは禁止。

## やらないこと(失敗後パターンを先に採らない)

- Olive型の「後で全書き直し前提で焼く」
- OpenCut型の「境界は後で入れる」
- Natron型の「所有権/キャッシュは後で直す」
- 「とりあえずserdeして、意味は後」

復旧手段(migration枠・Legacy variant・解凍手続き)は**予防が破れたとき**の出口として残す([rework-prior-art](2026-07-12-rework-prior-art.md)の復旧節)。日常の第一選択にしない。

## エージェントの着手前チェック(短い)

実装PRを開く前に Yes が全部そろうこと。1つでも No なら仕様改訂または依存待ち。

1. 焼く意味は仕様に表/宣言があるか
2. 追加的変更か(解釈変更ではないか)
3. 並列レーン依存を守っているか
4. 未決・ユーザー決定待ちに依存していないか
5. 完了条件に「意味の審判」(拒否テスト/意味論ゴールデン)があるか
