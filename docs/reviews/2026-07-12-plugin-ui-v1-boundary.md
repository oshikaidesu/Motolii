# 決定: v1プラグインUI境界は`NodeDesc`自動生成のみ(2026-07-12)

ステータス: **確定**(ユーザー承認)

対象仕様: [M3-ui-integration.md](../specs/M3-ui-integration.md)「プラグインパネルの拡張方式」  
先行レビュー: [prior-art-gap-survey](2026-07-12-prior-art-gap-survey.md) / [counter-review](2026-07-12-prior-art-gap-counter-review.md) / [M3-M4-gate-ledger](2026-07-12-M3-M4-gate-ledger.md)

## 最終判定

> **v1の安定公開境界は`NodeDesc`自動生成パネルのみ。**  
> `.slint`とwgpu自由描画は「技術的に可能だが、第三者向け契約として未成立」のため延期する。  
> 宣言レイアウトやギズモも、ファーストパーティ実験を経てv1.xまたはv2で解凍判断する。  
> 将来拡張しても、標準パネルだけで全パラメータを操作できることを不変条件とする。

## 補強サーチで確認した事実(判定は変わらず、根拠が厚くなった)

1. **Slint interpreter**: `.slint`の実行時コンパイルは正式提供。ただし通常経路は事前コンパイルであり、実行時ロードではfile loader・importパス・診断をホストが管理する必要がある。「ロードできる」ことと「第三者プラグインを安全・互換的に実行できる」ことは別。
2. **AE Effect UI & Events**: 標準パラメータとは別に、プラグインが描画・イベント処理を担当する明確な拡張口(パネル内UIとコンポジション上UI)。v1で同契約を開くと、同程度のイベント・入力・描画契約を背負う。
3. **Blender UILayout**: アドオンがホスト所有部品(`row`/`column`/`prop`等)でレイアウトを組む中間案は実在する。将来の宣言レイアウトは有力だが、現行`NodeDesc`とは別の公開語彙であり、v1へ急ぐ必要はない。
4. **wgpu / WebGPU**: 安全なRust APIとWGSL検証はあるが、第三者コードのGPU資源隔離にはならない。WebGPU仕様自体がGPUメモリ・計算資源によるDoSを実装側制限の問題として扱う。

## スコープ外にした調査

当初候補の「有料プラグイン5〜10件の能力コーパス」は、**v1縮小判断の前提には不要**。v2でどのUI拡張を開けるかを決める際の調査へ移す([backlog](../backlog.md) V2-8)。追加サーチを続けてもv1確定の精度はほぼ上がらない。
