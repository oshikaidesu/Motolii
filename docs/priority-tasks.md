## P0 — 絶対ブロッカー

### P0-1. 【解決済 2026-07-07】UI↔GPU の合成方式 → 完全ネイティブUIを採用しWebView経路を廃止
- **旧論点**: WebView UI とネイティブ(wgpu)描画層の合成を「別サーフェス重ね合わせ(overlay)」か「IPC経由テクスチャ転送」のどちらで行うか。overlay はOSごとにz順・リサイズ・DPI・透過・同期が破綻しやすく(特にLinux/WebKitGTKで致命的)、IPC転送は `GPU→CPU→IPC→CPU→GPU` バウンスが「軽さ」目的と矛盾する、という二律背反だった。
- **決定**: **Web版を作らず完全ネイティブアプリとし、UIも描画も同一ネイティブプロセス内で wgpu を直接使う。WebView/ブラウザWebGPU を描画経路に一切使わない。** これにより「UI↔GPUのIO経路」自体が存在しなくなり、合成方式の二択問題および入出力遅延が消滅する。
- **参照前例**: フルネイティブGPU UIは AviUtl2(DirectX11.3ネイティブ)、OpenCutデスクトップ(GPUI/Rust)、Zed(GPUI)で実証済みの構成。透過WebView overlay を採る主要動画ツールは確認できなかった(`docs` 調査より)。
- **残タスク**: 合成方式PoCは不要になったが、代わりに **P0-3(ネイティブUIフレームワーク選定)** が発生。

### P0-3. ネイティブRust GUIフレームワークを選定する(P0-1決定に伴う新規)
- **候補**: GPUI(Zed製、wgpu直描画・実運用実績あり/APIは発展途上) / egui(即時モード、導入容易だが凝ったUIは苦手) / iced(Elm風、retained) / Slint / Dioxus(native renderer) / 生の `winit` + wgpu 自前UI。
- **判断軸**: (1) wgpu描画サーフェスと同一フレーム内で自前レンダ結果を合成できるか、(2) タイムライン等の複雑UIウィジェットの表現力、(3) クロスプラットフォーム範囲(P1-2と連動)、(4) APIの安定性/破壊的変更リスク。
- **DoD**: 採用フレームワークと、「自前wgpu描画結果をUIのプレビュー領域に同一プロセス・ゼロコピーで表示する」最小PoCの成功。

### P0-2. 最小垂直スライス(MVP)の内容を1文で確定する
- **論点**: 「1本の動画を読み込み→色トラッキング→プロシージャルオーバーレイ生成→合成→1フレーム書き出し」で確定するか(`discussion-log` 未決定事項)。
- **なぜ最優先**: スライスが決まらないと P1 以降の技術選定・境界設計の判断基準が定まらない。プラグイン境界の早期凍結(下記の回避対象)を防ぐアンカーになる。
- **DoD**: 入力素材・各ステージの最小仕様・出力形式(単一PNG等)・成功判定を文章で確定し合意する。

## P1 — 早期確定必須

### P1-1. トラッキング解析の GPU/CPU 境界を決める
- **論点**: OpenCV.js(WASM) / ネイティブOpenCV(Rustバインディング) / 自前wgpu compute のいずれか。
- **リスク**: ネイティブOpenCVはビルド/リンクが重くクロスプラットフォームで苦労する。`GPUテクスチャ→OpenCV(CPU)→GPU` の往復は 5-2 で禁じた CPUバウンスを解析層で自ら踏むことになる。
- **DoD**: 採用方式と、フレームがどの層(GPU/CPU)を通るかのデータフロー図を確定。

### P1-2. ターゲットOS範囲を決める(Windows優先 vs 最初からクロスプラットフォーム)
- **リスク**: wgpuはバックエンド(Vulkan/Metal/DX12)ごとにバグ挙動が異なり、初期アーキとCI構成が変わる。
- **DoD**: v1のサポートOSと、それに伴うwgpuバックエンドの優先順位を確定。

### P1-3. プラグイン境界/traitの「凍結タイミング」を明文化する
- **論点**: 並列AIエージェント開発のため境界を早期凍結したい(`discussion-log` 未決定事項)が、要件未確定の段階での凍結は後方互換負債を最初に作る。
- **方針(提案)**: 境界凍結は **P0-2 の垂直スライスを1本通した後** に行う、と明示的に位置づける。それまではコア内部の一枚岩実装を許容する。
- **DoD**: 「凍結の前提条件(何が実証できたら凍結してよいか)」をドキュメント化。
